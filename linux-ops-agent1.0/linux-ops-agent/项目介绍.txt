================================================================================
          Linux Ops Agent — 项目介绍
================================================================================

一、项目简介
================================================================================

Linux Ops Agent 是一个基于 DeepSeek 大模型的智能 Linux 运维诊断系统。用户只需
用自然语言描述问题（如"检查磁盘空间"、"分析 SSH 登录失败"），系统即可自动识别
意图，在受控环境中执行预设的诊断脚本，最终生成结构化的 Markdown 诊断报告。

与直接让 AI 生成 Shell 命令不同，本系统采用"AI 选技能，脚本干实活"的安全架构：
DeepSeek 只负责意图理解和报告总结，实际执行的命令全部来自预先审计的受控脚本，
从根本上杜绝了 AI 误生成危险命令的风险。

================================================================================

二、服务对象
================================================================================

1. Linux 服务器管理员
   日常维护大规模集群，需要快速掌握多台服务器健康状态并及时定位异常。

2. 初中级后端开发者
   对 awk、sed、ss 等复杂命令掌握不深，但在开发部署中需要快速排查服务器问题。

3. 实验室管理员
   需定期对计算节点进行健康巡检，自动生成报告归档留痕。

================================================================================

三、解决的实际问题
================================================================================

痛点一：系统异常难以快速发现
  /var/log 等日志目录持续膨胀导致磁盘爆满，手动逐台排查耗时耗力，往往服务
  中断后才发现问题。本系统一键执行磁盘检查，自动定位大文件和空间瓶颈。

痛点二：运维命令门槛高
  排查进程、网络、日志需要组合 ps、awk、grep、ss 等命令，对初学者不友好。
  本系统将这些命令封装为预设脚本，用户用自然语言即可触发。

痛点三：重复性劳动多
  每日巡检需要在多台机器上重复执行相同命令并手动整理报告。本系统一条命令
  完成全部检查，并自动生成可存档的 Markdown 报告。

痛点四：AI 直接生成命令有安全风险
  直接从 AI 获取 Shell 指令可能包含 rm -rf、shutdown 等危险操作。本系统只
  执行预先审计的脚本，AI 不参与命令生成。

================================================================================

四、核心功能
================================================================================

（1）磁盘空间与 inode 检查
    自动检测分区使用率、inode 占用、/var/log 目录大小和超大文件定位。

（2）系统日志与 SSH 安全分析
    分析认证失败记录，统计暴力破解源 IP TOP 10，检测系统级错误和告警。

（3）CPU、内存与进程状态检查
    查看系统负载、内存使用、资源占用 TOP 10 进程及进程状态分布统计。

（4）网络与端口连通性检查
    检测 IP 地址、路由表、监听端口，测试 IP 层和 DNS 层连通性。

（5）一键综合巡检报告
    同时执行以上四项检查，利用 DeepSeek 汇总为包含问题概述、关键证据、
    可能原因和处理建议的专业诊断报告。

（6）自然语言交互
    支持中文自然语言输入，LLM 不可用时自动回退本地关键词匹配，无需 API Key
    也能正常使用。

================================================================================

五、技术架构
================================================================================

系统采用"规划 → 执行 → 报告"三阶段管道架构：

  用户自然语言输入
       │
       ▼
  ┌─────────────────┐
  │  Planner 意图路由 │  DeepSeek 语义理解 / 本地关键词回退
  │  选择匹配的 Skill │  危险操作正则拦截（rm -rf、shutdown 等）
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │ Executor 安全执行│  路径校验（脚本必须在 scripts/ 目录下）
  │  运行 Bash 脚本  │  超时保护、命令白名单、高风险 Skill 禁用
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │  Report 报告生成 │  DeepSeek 结构化总结 / 本地模板回退
  │  Markdown 输出   │  包含证据、原因、建议的完整诊断报告
  └─────────────────┘

技术栈：Python 3.10+ / Bash / YAML / DeepSeek API（OpenAI 兼容协议）

================================================================================

六、项目目标
================================================================================

1. 自动意图路由：准确解析用户自然语言需求，自动匹配并分发到合适的诊断脚本。
2. 自动化系统检测：覆盖磁盘、网络、进程、日志四大运维核心场景。
3. 自动生成结构化报告：利用 LLM 将命令输出整理为包含处理建议的 Markdown 报告。
4. 确保极高安全性：绝不执行 AI 生成的命令，所有操作来自预先审计的受控脚本。
